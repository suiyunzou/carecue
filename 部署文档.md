# 阿里云服务

### 1. 登录并准备目录

登录阿里云服务器终端后，创建一个存放项目的目录并进入：

```bash
# 切换到 root 目录（或者 /opt 目录）
cd /opt

# 拉取你 GitHub 上的代码
git clone https://github.com/suiyunzou/carecue.git

# 进入项目目录
cd carecue
```

### 2. 第一次运行一键部署脚本

脚本会自动帮你配置 2G 虚拟内存、安装 Docker，并准备环境变量文件：

```bash
# 赋予脚本执行权限
chmod +x deploy.sh

# 以管理员权限运行
sudo ./deploy.sh
```

> **注意：** 此时脚本会停下来，提示你“自动生成了 `.env` 文件，但需要填入 API Key”。

### 3. 配置你的私密 API Key

修改刚刚生成的 `.env` 文件，填入真实密钥，并配置你的公网 IP 以允许跨域：

```bash
# 打开文本编辑器修改配置
nano .env
```

找到下面这三行并进行修改：

```env
OPENROUTER_API_KEY="sk-or-v1-xxxxxxxxx"
FIRECRAWL_API_KEY="fc-xxxxxxxxx"
# 将你的阿里云服务器公网 IP 加入到允许跨域的列表中（注意加上 http:// 前缀，如果有多个用逗号隔开）
CLIENT_ORIGIN="http://localhost:5173,http://127.0.0.1:5173,http://182.92.6.98"
```

*修改完毕后，按* *`Ctrl+O`* *再按* *`Enter`* *保存，按* *`Ctrl+X`* *退出编辑器。*

### 4. 再次运行脚本完成部署

配置好 Key 后，再次运行部署脚本，它会自动构建并启动所有服务：

```bash
sudo ./deploy.sh
```

> 这个过程在 2核 2G 机器上大约需要 3-5 分钟，请耐心等待，直到看到 `部署流程执行完毕` 和你的访问网址。

### 5. 访问与网关配置

部署成功后，你还需要在阿里云控制台开放相应端口，才能通过公网 IP 访问应用。

**步骤 1：获取公网 IP**
在阿里云服务器控制台实例列表中，找到你服务器的 **“公网 IP”** 地址。

**步骤 2：配置安全组（放行端口）**
CareCue 前端容器映射到了宿主机的 **80 端口**。你必须在阿里云的安全组中开放 80 端口：
1. 登录阿里云控制台，进入 **ECS 实例详情页**。
2. 点击 **“安全组”** -> **“配置规则”**。
3. 点击 **“手动添加”**，配置如下：
   - **目的**：`80/80`
   - **源**：`0.0.0.0/0`（允许所有 IP 访问）
   - **协议**：TCP
4. 保存规则。

**步骤 3：浏览器访问**
打开浏览器，直接在地址栏输入你的服务器公网 IP：
```
http://你的服务器公网IP
```
即可看到 CareCue 的前台页面。

*(注：如果你后续需要配置 HTTPS 或自定义域名，可以在阿里云控制台申请免费 SSL 证书，并配置 Nginx 反向代理或使用阿里云的负载均衡 SLB，目前测试阶段直接使用 80 端口和 IP 访问即可。)*

### 日常运维备用命令清单（收藏备用）

部署成功后，如果你需要管理服务或更新代码，可以使用以下命令（都需要在 `carecue` 目录内执行）：

- **查看实时运行日志**（排查报错用）：
  ```bash
  docker compose logs -f
  ```
  *(按* *`Ctrl+C`* *退出查看)*
- **拉取最新代码并重新部署**（你在本地更新后）：
  ```bash
  git pull
  sudo ./deploy.sh
  ```
- **停止并下线所有服务**：
  ```bash
  docker compose down
  ```
- **重启服务**（如果卡死了）：
  ```bash
  docker compose restart
  ```

### 常见问题排查 (FAQ)

#### 1. Docker 拉取镜像超时 (如拉取 `postgres:17-alpine` 失败)
如果在部署过程中遇到 Docker 无法从 Docker Hub 下载镜像（提示 `registry-1.docker.io` 连接超时），可以通过配置国内镜像加速器解决：

```bash
# 1. 创建或编辑 docker 配置文件
sudo mkdir -p /etc/docker
sudo nano /etc/docker/daemon.json
```

将以下内容粘贴进去并保存（按 `Ctrl+O`，`Enter` 确认，再按 `Ctrl+X` 退出）：
```json
{ 
  "registry-mirrors": [
    "https://docker.mirrors.sjtug.sjtu.edu.cn",
    "https://dockerpull.com",
    "https://docker.m.daocloud.io"
  ] 
}
```

> **💡 强烈建议（针对阿里云服务器）：** 
> 国内公共镜像源近期大面积关停或不稳定。最稳定的方案是使用你自己的“阿里云专属加速器”：
> 1. 登录阿里云控制台，搜索 **“容器镜像服务 (ACR)”**。
> 2. 左侧导航栏找到 **“镜像工具” -> “镜像加速器”**。
> 3. 复制你的专属加速器地址（例如 `https://xxxxxx.mirror.aliyuncs.com`），将其替换到上述 `daemon.json` 的配置中。

然后重启 Docker 服务，并再次运行部署：
```bash
# 2. 重载配置并重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker

# 3. 重新执行部署脚本
sudo ./deploy.sh
```
