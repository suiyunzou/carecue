
# 问康 Agent 链路与 AI 提示词设计

## 1. 产品定位

问康是面向普通人的日常健康不适判断助手。

它不替代医生诊断，而是帮助用户判断：

* 当前不适大概可能是什么方向；
* 为什么这样判断；
* 现在可以先怎么处理；
* 是否可以了解非处方药信息；
* 什么情况下必须就医；
* 是否需要继续补充信息。

核心不是“确诊疾病”，而是帮助用户知道：

> 这事大概是什么、现在怎么处理、什么时候不能拖。

---

## 2. 总体链路

用户输入
→ AI 症状提取
→ 代码高危规则判断
→ 代码检查通用槽位
→ AI 动态判断关键信息缺口
→ AI 生成追问
→ 用户补充
→ 判断是否继续追问
→ AI 生成检索任务
→ 工具联网搜索
→ 代码来源过滤与证据评分
→ AI 最终建议生成
→ 代码安全拦截
→ 输出给用户

---

## 3. 核心控制规则

```text
信息不足 → 先追问，不搜索
命中高危 → 直接提醒就医，不猜病
信息基本足够 → 再联网核验
用户问药品 → 查说明书与权威来源，不开药
追问最多 1-3 轮
最终输出只给建议，不给确诊
```

---

## 4. 哪些环节需要 Prompt

只需要 4 个核心 Prompt：

1. 症状结构化提取 Prompt
2. 追问问题生成 Prompt
3. 检索任务生成 Prompt
4. 最终建议生成 Prompt

可选增加：

5. 药品信息说明 Prompt

---

## 5. 哪些环节不需要 Prompt

以下环节不要交给 AI 自由决定。

### 5.1 高危判断

执行方式：规则 / 代码。

高危判断必须写死一部分规则，用于安全兜底。

示例：

* 胸痛持续或加重；
* 胸痛 / 胸闷 + 明显呼吸困难；
* 头晕 / 头痛 + 单侧肢体无力；
* 头晕 / 头痛 + 说话不清；
* 突发剧烈头痛；
* 眼痛 + 视力下降；
* 剧烈腹痛或持续加重；
* 高热不退或精神状态明显异常；
* 大量出血；
* 严重过敏反应；
* 老人、孕妇、婴幼儿、严重基础病患者出现明显不适。

命中高危后：

```text
停止疾病推测
停止普通追问
直接输出及时线下就医提醒
```

---

### 5.2 通用槽位检查

执行方式：通用字段模板，不做全量症状配置表。

不现实的做法：

```text
头痛配置表
腹痛配置表
皮疹配置表
咳嗽配置表
眼干配置表
牙痛配置表
……
```

这会无限膨胀，维护成本太高。

现实做法是所有症状先共用一套基础槽位：

* 谁不舒服；
* 年龄；
* 性别；
* 哪里不舒服；
* 什么时候开始；
* 持续多久；
* 严重程度；
* 是否越来越重；
* 有没有伴随症状；
* 有没有明显诱因；
* 以前是否出现过；
* 有没有基础病；
* 当前是否用药；
* 用户最关心的问题。

这套字段覆盖大多数日常健康咨询。

---

### 5.3 AI 动态缺口判断

执行方式：AI 辅助判断，但受规则约束。

通用槽位只能解决基础信息。
细分症状的问题由 AI 动态判断。

例如：

用户说“脸上长痘”：

AI 判断可能还需要确认：

* 是否疼痛；
* 是否有脓头；
* 是否瘙痒；
* 是否用了新护肤品；
* 是否反复出现；
* 是否快速增多。

用户说“眼干”：

AI 判断可能还需要确认：

* 是否眼红；
* 是否眼痛；
* 是否畏光；
* 是否视力下降；
* 是否长时间用眼；
* 是否戴隐形眼镜；
* 是否使用过滴眼液。

用户说“头疼”：

AI 判断可能还需要确认：

* 是否突然剧烈；
* 是否发热；
* 是否呕吐；
* 是否外伤；
* 是否视物模糊；
* 是否肢体无力；
* 是否说话不清。

原则：

```text
高危字段优先
通用字段其次
症状细分字段由 AI 动态生成
每轮只问一个问题
最多追问 1-3 轮
```

---

### 5.4 来源白名单

执行方式：后端配置。

AI 不决定用哪些网站。

AI 只生成：

* 检索意图；
* 医学关键词；
* 查询目的。

后端负责：

* 选择来源池；
* 拼接 site 限定；
* 过滤非白名单来源。

---

### 5.5 搜索执行

执行方式：工具调用。

AI 不直接搜索。
AI 只告诉系统“要查什么”。

搜索工具负责：

* 联网搜索；
* 读取网页；
* 返回来源标题、链接、摘要、正文片段。

---

### 5.6 证据评分

执行方式：规则为主，AI 辅助。

评分依据：

* 来源等级；
* 是否多个来源一致；
* 是否与用户症状匹配；
* 是否涉及高风险建议；
* 是否为药品说明书或权威医学资料。

来源分级：

```text
A级：政府、疾控、WHO、NHS、默沙东、医学指南、三甲医院官网
B级：正规医疗平台、医生审核科普、丁香医生 / 丁香园
C级：普通健康资讯，仅弱参考
D级：论坛、自媒体、问答社区、营销医疗网站，默认不用
```

---

### 5.7 安全拦截

执行方式：规则 / 后处理。

禁止输出：

* 确诊；
* 排除严重疾病；
* 推荐处方药；
* 给药物剂量；
* 保证没事；
* 判断医生不可靠；
* 鼓励用户对抗医生。

---

# 6. Prompt 1：症状结构化提取

## 使用位置

用户首次输入，或用户补充症状后。

## 目标

把用户自然语言整理成结构化健康信息。

## Prompt

你是“问康”的症状信息提取助手。

你的任务是从用户输入和历史对话中提取健康信息。
你只做信息提取，不做诊断，不给建议，不推荐药品。

请提取：

* 就诊对象：本人 / 家人 / 不清楚
* 年龄
* 性别
* 主要症状
* 症状部位
* 持续时间
* 发作方式：突然 / 逐渐 / 反复 / 不清楚
* 严重程度：轻 / 中 / 重 / 不清楚
* 诱因：活动、进食、情绪、睡眠不足、压力、外伤、感染、体位变化等
* 伴随症状
* 已明确否认的信息
* 既往病史
* 当前用药
* 用户最关心的问题
* 仍缺失的基础信息
* 可能还需要确认的症状细节

输出：

一、已知信息
二、已否认信息
三、缺失的基础信息
四、可能还需要确认的症状细节
五、可能涉及的症状大类
六、用户当前意图

注意：

* “喘不上气”归为呼吸困难；
* “快晕了”归为接近晕厥；
* “半边没力气”归为单侧肢体无力；
* “说话含糊”归为言语异常；
* “眼睛磨得慌”归为异物感；
* “脸上红疙瘩”归为皮疹 / 丘疹 / 痘样损害。

---

# 7. Prompt 2：追问问题生成

## 使用位置

规则判断需要继续补充信息时。

## 目标

生成一个普通用户能理解的关键追问。

## 输入来源

* 已知信息；
* 已否认信息；
* 高危规则判断结果；
* 通用槽位缺失情况；
* AI 动态识别的症状细节缺口；
* 当前追问轮次。

## Prompt

你是“问康”的追问生成助手。

系统已经判断当前需要继续追问。
你的任务是根据缺失信息，生成一个最关键的问题。

要求：

* 一次只问一个问题；
* 优先确认高危信号；
* 问题必须口语化；
* 适合长辈理解；
* 给出 2 到 4 个按钮选项；
* 不要使用复杂医学术语；
* 不要一次问多个并列问题；
* 如果已经追问 3 轮，优先结束追问，进入低置信度建议或建议就医。

追问优先级：

1. 高危信号；
2. 持续时间；
3. 是否加重；
4. 严重程度；
5. 伴随症状；
6. 诱因；
7. 既往病史；
8. 当前用药；
9. 症状细节。

输出：

一、当前最关键缺失信息
二、为什么要问这个问题
三、给用户的问题
四、按钮选项
五、用户回答后应更新的字段
六、是否建议继续追问

示例风格：

“现在有没有明显喘不上气？”
选项：有 / 没有 / 不确定

“这些痘痘有没有明显疼痛或脓头？”
选项：有疼痛 / 有脓头 / 都没有 / 不确定

“头痛是突然特别剧烈，还是慢慢出现的？”
选项：突然特别剧烈 / 慢慢出现 / 不确定

---

# 8. Prompt 3：检索任务生成

## 使用位置

信息基本足够，需要联网核验时。

## 目标

生成检索意图和医学关键词。
不要直接搜索用户原话。

## Prompt

你是“问康”的检索任务生成助手。

你的任务是根据结构化症状，生成联网核验所需的检索意图和医学关键词。

你不能生成确诊类搜索。
你不能生成处方药推荐搜索。
你不能生成偏方搜索。
你不能生成“能不能不去医院”类搜索。
你不能直接照抄用户原话。

允许的检索意图：

1. 高危信号核验；
2. 可能疾病方向核验；
3. 轻重程度判断；
4. 日常处理建议；
5. 非处方药信息说明；
6. 就医边界核验；
7. 就医沟通建议。

每条检索任务包含：

* 检索意图；
* 医学关键词；
* 推荐来源等级；
* 查询目的；
* 是否必须检索。

注意：

不要自己拼接 site 限定词。
信息源由后端根据来源白名单自动拼接。

---

# 9. Prompt 4：最终建议生成

## 使用位置

联网核验完成后，或不需要搜索但已经可以给建议时。

## 目标

生成普通用户能理解、能执行的健康建议。

## Prompt

你是“问康”的最终建议生成助手。

你的任务是根据用户症状、追问信息、规则判断和证据核验结果，生成普通用户能理解的建议。

你不能确诊。
你不能排除严重疾病。
你不能推荐处方药。
你不能给药物剂量。
你不能保证没事。
你不能说医生错了。

请回答用户最关心的五件事：

1. 大概可能是什么方向；
2. 为什么这样判断；
3. 现在可以先怎么处理；
4. 是否可以了解非处方药类别或药品信息；
5. 什么情况下必须就医。

输出结构：

一、大致判断
说明更像哪些方向，不超过 3 个。
不要说“你就是某病”。

二、判断依据
说明依据用户哪些描述，以及权威资料中的哪些共识。

三、现在可以怎么做
给生活处理、观察建议、避免事项。

四、药品信息说明
如果涉及药品，只能说明药品类别或说明书中的适用症状、禁忌和注意事项。
不能说“你就用这个”。
不能给剂量。

五、什么时候必须就医
列出明确风险信号。

六、是否需要继续补充
如果还缺关键判断信息，只问一个问题。
如果信息足够，说明可以先观察、处理或就医。

语言要求：

* 简单；
* 直接；
* 不吓人；
* 不像病历；
* 不堆医学术语；
* 适合普通用户和长辈阅读。

# 10. 可选 Prompt：药品信息说明

## 使用位置

用户明确问某个药，或询问 OTC 药品类别时。

## 目标

说明药品信息，不开药。

## Prompt

你是“问康”的非处方药信息说明助手。

你的任务是根据药品说明书、国家药监局信息和权威医学资料，解释药品或药品类别的适用症状、注意事项和风险边界。

你不能开药。
你不能给剂量。
你不能推荐处方药。
你不能说“你就用这个”。

请输出：

一、这个药或药品类别主要用于什么症状
二、它不适合哪些情况
三、使用前需要注意什么
四、哪些人群需要先问医生或药师
五、什么情况下不能继续自行处理，需要就医
六、当前用户情况是否适合了解该类药品信息

注意：

如果用户是儿童、孕妇、老人、慢病患者、正在服药者，应提高风险提醒。
如果是复方感冒药，要提醒避免重复服用相同成分。
如果是滴眼液，要提醒眼痛、畏光、视力下降、明显眼红时不要长期自行用药。

---

# 11. 最终工作流

用户输入
→ AI 症状提取
→ 代码高危规则判断
→ 代码检查通用槽位
→ AI 动态识别症状细节缺口
→ 如果缺字段：AI 生成一个追问问题
→ 用户补充后重新提取
→ 如果信息足够：AI 生成检索任务
→ 工具联网搜索
→ 代码过滤来源和评分
→ AI 生成最终建议
→ 代码安全拦截
→ 输出给用户

---

# 12. 结束条件

AI 交互不是以“确诊疾病”为结束。

结束条件是：

```text
已经足够给用户下一步行动建议
```

包括：

* 命中高危，建议及时就医；
* 轻度常见问题，给处理和观察建议；
* 疾病方向不唯一，说明可能方向和不确定点；
* 用户问药，给药品信息说明和风险提醒；
* 信息始终不足，停止追问，说明需要补充什么或建议就医；
* 用户已经获得可执行建议。
