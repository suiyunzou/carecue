# 模块：M4 用户测试与优化

> 对应《CareCue 4.0 设计文档》第三部分 M4 与第五部分成功标准。物理位置 `server/src/`。

## 职责

用可重放的模拟咨询批量验证系统，并量化设计文档第五部分的 5 条成功标准；发现并修复 bug。

## 核心逻辑

`m4.test.ts` 构建确定性咨询 harness（MockLlm 驱动，跨 10 组症状）：
- 10 例正常咨询：逐轮回答阴性 → 收敛非确诊报告；校验无禁语、R0、含护理+就医建议。
- 10 例急症咨询：首轮即针对高危红旗，回答其阳性信号 → 触发 emergency（R3）。
- 边界：工具失败一次仍继续；模糊描述不崩溃。
- 用 `MemoryTracer` 统计每次咨询的非强制（=LLM）决策次数与轮均耗时。

## 成功标准核对（设计文档第五部分）

| 标准 | 结果 |
|------|------|
| ① 端到端跑通真实案例并给出合理报告 | ✅ 20 例（10 正常 + 10 急症） |
| ② 任一工具失败 1 次对话仍继续 | ✅ flaky lookup 重试后继续 |
| ③ 高危红旗 100% 触发急症 | ✅ 召回 10/10 |
| ④ 单次咨询 LLM 调用 ≤ 8 次 | ✅ 实测最大 7 |
| ⑤ 平均响应时间 ≤ 8 秒/轮 | ✅ mock 下 <1ms（真实延迟取决于 LLM provider） |

## 测试用例（Few Shot）

`npm run test:m4`（5 用例全过）；`npm run test:agent4` 串跑 M1–M4 共 35 用例。

## 不做什么

- 真实用户招募与人工评估属线上运营，超出代码范围；这里用模拟咨询逼近成功标准。
- ⑤ 的真实响应时间需在配置真实 LLM 后用线上 trace（duration）观测，非本地可测。
