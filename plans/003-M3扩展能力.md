# 模块：M3 扩展能力

> 对应《CareCue 4.0 设计文档》第三部分 M3：扩展能力。物理位置 `server/src/`。

## 职责

把骨架扩成可用系统：知识库扩到 10 组症状，补全工具（extract_facts、hypothesis 系列、
search_medical 接 Firecrawl），落实 Guard 全部 5 条规则，可观测性日志落 PostgreSQL。

## 输入输出

- `extract_facts({ text })` → 写回 Workspace（症状/年龄/性别/时长）。由 `requiredAction` 在每轮开头强制。
- `add_hypothesis({ name, initialEvidence? })` / `update_hypothesis({ name, delta, evidence? })` → 维护假设权重。
- `search_medical({ query })` → 经 `SearchClient`（Firecrawl，可注入）检索，片段/来源写回 `searchResults`。
- Trace → `PrismaTraceSink` 落 `agent_traces`（每调用一行）与 `agent_workspaces`（每会话快照 JSONB）。

## 依赖

- 内部：M1/M2 全部 + `agent/extractor`、`agent/search`、`agent/trace`
- 外部库：`yaml`、`@mendable/firecrawl-js`（懒加载）、`@prisma/client`（落库，结构化注入）

## 核心逻辑

1. 知识库：`red_flags.yaml` / `care_plans.yaml` 扩到 10 组；新增 `symptom_synonyms.yaml`，
   `loader.matchSymptoms(text)` 用「最长优先」同义词匹配识别口语（如「肚子疼」→腹痛）。
2. `RuleExtractor`（确定性，默认/降级）：同义词症状 + 年龄/性别/时长正则。可注入 LLM 抽取器替换。
3. 主循环：`requiredAction` 新增「本轮未抽取 → 强制 extract_facts」；M1 最小集（无该工具）时
   由引擎用抽取器直接种子，保持兼容。
4. Guard 5 条：①症状未加载强制 lookup、⑤高危 positive 强制急症（`requiredAction`）；
   ②pending 禁报告、③重复问题（`guard`）；④报告禁语（`guardReport`）。新增「红旗已加载禁重复检索」防空转。
5. 可观测性：`MemoryTracer` 可挂 `TraceSink`；`PrismaTraceSink` 把 snapshot→workspaces、其余→traces。

## 测试用例（Few Shot）

`npm run test:m3`（13 用例全过）：10 组覆盖与首轮加载、口语抽取、hypothesis 权重 clamp、
search 写回/优雅失败、search 失败仍继续、Guard 5 条逐条、PrismaTraceSink 落库。

## 不做什么

- 不在测试中真连 Firecrawl/PG（注入假实现；migration 已写，部署时 `prisma migrate deploy`）
- 不做症状域动态重分类（后续轮次新增症状不重跑 lookup）—— 留待后续
- 不做 RAG / 向量库（设计文档 4.1 明确不做）
