# 模块：M1 约束式事件循环骨架

> 对应《CareCue 4.0 设计文档》第三部分 M1：骨架打通。
> 物理位置：`server/src/`（全新干净实现，不改动遗留 `server/agent/`，待 M2~M4 验证后再迁移/删除）。

## 职责

用「约束式事件循环」打通一次咨询的最小闭环：用户描述不适 → 加载红旗 → 逐条排查 → 收尾出报告或触发急症提示。本里程碑用 Mock LLM 驱动决策，知识库只覆盖一组症状（头晕 + 胸闷）。

## 输入输出

- 输入：`runConsult({ workspace, userMessage, llm, knowledge })`
  - `workspace`：跨轮持久的共享工作区（M1 用内存 Map）
  - `userMessage`：本轮用户自然语言输入
- 输出：`ConsultResponse`
  - `followup`：向用户追问一个问题（中断循环，等下一轮）
  - `final_report`：最终非确诊报告（护理建议 + 就医提示 + 免责声明）
  - `emergency`：检测到高危红旗 positive，急症提示

## 依赖

- 内部：`schemas`（Zod 类型）、`knowledge/loader`（YAML）、`agent/workspace`、`agent/guard`、`agent/llm`、`tools/*`
- 外部库：`yaml`（解析知识库）、`zod`（入参出参强约束）

## 核心逻辑（约束式事件循环，对应设计文档 2.1）

```
每轮 runConsult：
  1. rounds++，naive 种子症状（M1 临时；extract_facts 在 M3 替换）
  while step < MAX_STEPS:
    2. 硬约束：requiredAction(ws)  —— 症状已知但红旗未加载 → 强制 lookup_red_flags；
                                     高危红旗 positive → 强制 generate_report(急症)
    3. 决策：forced ?? llm.decide(summary, lastUserMessage, tools, feedback)
    4. Guard：guard(action, ws) —— 拒绝则把 reason 反馈给 LLM 重新决策（不抛错）
    5. 执行工具（失败重试 1 次，再失败把错误作为工具结果反馈给 LLM）
    6. 结果写回 Workspace（工具内增量更新）
    7. 终止判断：
         interrupt(ask_user)  → 返回 followup
         emergency            → 返回 emergency
         report               → guardReport(文本) 通过 → 返回 final_report，否则要求重写
```

不是状态机：没有「第几步」，每轮独立决策，约束由 Guard 兜底。

## Guard V1 规则（M1 落地前 4 条 + 第 5 条急症）

1. 症状非空但红旗未加载 → 强制 lookup_red_flags（由 requiredAction 实现）
2. 有 pending 红旗 → 禁止 generate_report（但高危 positive 时放行急症）
3. 重复问题 → 拒绝 ask_user
4. 报告含「确诊 / 一定是 / 必须服用 X mg」→ 拦截要求重写（guardReport）
5. 高危红旗 positive → 强制急症提示（由 requiredAction + generateReport 实现）

## 测试用例（Few Shot）

1. **正常排查（全部排除）**：用户「头晕 + 胸闷」，逐轮回答均无危险信号 →
   终态 `final_report`，含护理建议 + 就医建议，且**不含**任何被禁短语。
2. **急症（心源性 positive）**：用户回答「压榨性疼痛、放射到左臂、冒冷汗」→
   终态 `emergency`，`triggeredCombination` 含「心源性」。
3. **红旗强制加载**：症状已知但未加载红旗时，第一步**必然**是 lookup_red_flags
   （即便 Mock LLM 想跳过，requiredAction 也会强制）。
4. **重复问题拦截（边界）**：把同一问题塞进 askedQuestions 后再 ask_user，
   guard 返回 `allow:false`。

## 不做什么

- 不接真实 LLM（M2）、不接 Firecrawl（M3）、不落 PG（M3 可观测性）
- 不实现 extract_facts / hypothesis 系列工具（M3）
- 不扩展知识库到多组症状（M3）
- 不改动遗留 `server/agent/` 与现有 Express 路由（迁移在后续里程碑）
